# Project Summary - SteelDoorAi Demo

> A concise reference for anyone (or any AI assistant) picking up this project to extend it.

## What this is

An AI-assisted quoting assistant for a steel / security / fire door business. A natural-language
chatbot collects requirements and drives a **deterministic, auditable quoting engine**. Exposed as
a small FastAPI service with a browser chat UI.

- **Built by:** Kieran (co-authored by GitLab Duo)
- **Created:** 2026-06-10 (UTC)
- **Workflow:** direction (Kieran) -> scaffold/codegen (GitLab Duo) -> VS Code -> DeepSeek audit -> final pass (Opus / Fable)

## Core design principle

**The LLM handles conversation; the deterministic engine handles every number.** Prices are never
generated by a model. This keeps quotes correct, repeatable, and auditable - essential for
fire-rated, compliance-sensitive products.

## How it fits together

| Layer | File | Responsibility |
|-------|------|----------------|
| Web UI | `app/static/index.html` | Browser chat front end |
| API | `app/main.py` | FastAPI routes: `/api/chat`, `/api/quote`, `/health` |
| Orchestration | `app/chat.py` | NL extraction + reply; pluggable LLM (mock / DeepSeek) |
| Pricing | `app/quoting.py` | Deterministic, rule-based quote calculation |
| Lead capture | `app/store.py` | SQLite store for enquiries (`/api/enquiry`) |
| Schemas | `app/models.py` | Pydantic request/response models |
| Tests | `tests/` | pytest: quoting math, NL extraction, API endpoints |
| CI | `.gitlab-ci.yml` | Runs pytest on push / MR |
| Container | `Dockerfile` | Containerised run |
| Shortcuts | `Makefile` | `make install / test / demo` |

## Run it

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
pytest
uvicorn app.main:app --reload   # http://localhost:8000
```

## Current state

- Functional end to end with the mock LLM (zero config).
- DeepSeek drops in via `.env` (`LLM_PROVIDER=deepseek`, `DEEPSEEK_API_KEY=...`).
- **Product model reflects Steel Door Company's real range**: single/double sets, internal /
  external / fire-rated / wine-room, hinged or sliding, glass (transparent/opaque/reeded),
  any RAL colour, fixed side panels, plus the Summer Sale (10% off, capped at GBP 1,000).
- **Pricing anchors are real** (single from GBP 1,500, double from GBP 2,800, published on the
  site). All **uplift amounts are estimates** marked `TODO: confirm with Steel Door Company`
  in `app/quoting.py` - do not present them as official figures.
- CI pipeline shows "failed" only because the GitLab group has no CI runner minutes provisioned;
  zero jobs run. Tests pass locally.

## Further upgrades (prioritised backlog)

Ordered roughly by effort-to-value. Each item is a self-contained next step.

### Quick wins
1. **Replace placeholder pricing** with the real price list in `app/quoting.py` (`PRICING`).
2. **Persist quotes** - save each quote to a small SQLite/Postgres table; add a quote reference number.
3. **Email / PDF quote** - generate a branded PDF and email it (e.g. ReportLab + SMTP).
4. **Input clarification loop** - when the chatbot can't extract size/quantity, ask a follow-up
   question instead of giving the generic prompt.
5. **Structured extraction via LLM** - optionally replace the regex extractor with an LLM
   function-call that returns a validated `QuoteRequest`, with the regex as a fallback.

### Medium
6. **RAG support assistant** - ground answers in product specs, fire-rating certificates, and
   building-regulation references; return cited sources. (See RECOMMENDATIONS.md #2.)
7. **Document / spec extraction** - parse architect drawings and PDF schedules to auto-populate
   quotes; flag low-confidence fields for human review.
8. **CRM / ERP integration** - push qualified leads + quotes to the existing system; add auth,
   retries, idempotency keys, and an audit log.
9. **Auth + rate limiting** on the API before any public exposure.
10. **Evaluation harness** - a set of known-good prompts/specs asserted against expected quotes to
    catch regressions in extraction and pricing.

### Larger / strategic
11. **Image-based product suggestion** - upload a photo of an opening; vision model suggests door
    types/finishes, human confirms before quoting.
12. **Admin pricing UI** - let staff edit `PRICING` safely without code changes (with versioning).
13. **Observability** - request/latency/cost metrics, structured logging, and LLM call tracing.
14. **Multi-channel** - expose the assistant via the website widget, WhatsApp, and email intake.

### Engineering hygiene to add alongside
- Type checking (mypy) and linting (ruff) in CI.
- Pin and scan dependencies (Dependency Scanning).
- GDPR: data minimisation, retention policy, human-in-the-loop for customer-facing output.
