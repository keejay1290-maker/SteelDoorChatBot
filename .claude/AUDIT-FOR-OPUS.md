## 1. Security

### SEC-01 — SQL injection surface
**Files:** `app/store.py`, `app/main.py`
**Check:** Every `conn.execute()` call uses parameterised `?` placeholders — no string concatenation of user values.
**Pass:** Zero f-string or `+` concatenation in SQL strings. Confirm `save_enquiry`, `save_quote`, `save_llm_metric`, pricing writes all use tuples.

### SEC-02 — Auth bypass on dashboard + admin
**Files:** `app/main.py` — `_require_dashboard()`, all routes decorated with `Depends(_require_dashboard)`
**Check:** `/dashboard`, `/api/dashboard/*`, `/admin/pricing`, `/api/admin/pricing` all require auth. No route accidentally missing the Depends.
**Pass:** List every protected route and confirm Depends present. Check for any `.get`/`.post` that returns sensitive data without auth.

### SEC-03 — PII in logs
**Files:** `app/chat.py`, `app/main.py`, `app/hubspot.py`, `app/email_sender.py`
**Check:** `logger.*` calls must not log raw email addresses, phone numbers, or customer names at INFO level (only session_id + score).
**Pass:** No `%s` placeholder receives `s.email`, `s.phone`, or `s.name` in any `logger.info()` call. ERROR level is acceptable with truncation.

### SEC-04 — Rate limiting completeness
**Files:** `app/main.py` — `_limiter` application
**Check:** `/api/chat` and `/api/quote` have rate limit decorators. `/api/enquiry` should also be limited (form spam vector).
**Pass:** Confirm slowapi `@limiter.limit(...)` on chat, quote, enquiry. Check that `/api/dashboard/*` is NOT rate-limited (unnecessary + auth-protected).

### SEC-05 — Secrets in source
**Files:** `.env.example`, `app/main.py`, all `app/*.py`
**Check:** No hardcoded API keys, tokens, or passwords in any `.py` file. `.env` must not be committed.
**Pass:** `git log --all -- .env` returns nothing. No `gsk_` or `pat-eu1-` strings in any `.py`.

### SEC-06 — CORS origin whitelist
**Files:** `app/main.py` — CORSMiddleware config
**Check:** `allow_origins` is not `["*"]`. Should be driven by `ALLOWED_ORIGINS` env var with a sensible default.
**Pass:** Read the CORS setup. Confirm wildcard is not present in production path.

### SEC-07 — HubSpot token scope minimum
**Files:** `app/hubspot.py`
**Check:** The code only calls Contact POST + Deal POST. Verify it never reads contacts (which would need read scope).
**Pass:** No GET requests to HubSpot API. Only `contacts` and `deals` POST endpoints used.

## 3. Performance

**Pass:** All DB calls use `with _connect() as conn:` — no leaked connections.

### PERF-02 — LLM call on every chat message
**Files:** `app/chat.py` — `_llm_extract_fields()`
**Check:** Field extraction LLM call fires even when all fields are already collected. Verify there's a guard to skip extraction when readiness is already 100 or all required fields are present.
**Pass:** If no guard exists, flag as PERF issue (wastes tokens + adds ~800ms per message in late conversation).

### PERF-03 — Dashboard stats query efficiency
**Files:** `app/store.py` — `get_dashboard_stats()`
**Check:** Multiple `conn.execute()` calls in sequence — not batched. For Vercel SQLite this is fine for demo, but verify no N+1 on `sessions` table (no per-row queries).
**Pass:** All queries are aggregates or LIMIT 10. No loops calling `conn.execute()`.
